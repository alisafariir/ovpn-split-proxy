# App icons — format and sizes

Icons go in **`assets/icons/`**.

---

## 1. Development / runtime (window + tray)

| File        | Format | Size      | Use |
|------------|--------|-----------|-----|
| `icon.png` | PNG    | **256×256** (recommended) or at least 32×32 | Main app icon: window title bar, taskbar. The tray icon is generated by resizing this to 16×16. |

- **Path:** `assets/icons/icon.png`
- If `icon.png` is missing, the window uses the default Electron icon and the tray uses a small colored square.

---

## 2. Built exe / installer (electron-builder)

For the packaged Windows exe and installer, electron-builder can use:

| File       | Format | Sizes | Use |
|------------|--------|--------|-----|
| `icon.ico` | ICO    | **256×256**, 48×48, 32×32, 16×16 (one multi-size .ico) | Best on Windows: exe icon, installer, shortcuts. |
| or `icon.png` | PNG | **256×256** | electron-builder will convert to .ico for the build. |

- **Path:** `assets/icons/icon.ico` or `assets/icons/icon.png`
- To use a specific file in the build, set in `package.json` under `build.win`:
  ```json
  "win": {
    "icon": "assets/icons/icon.ico"
  }
  ```
  If you only have `icon.png`, use `"icon": "assets/icons/icon.png"`.

---

## Summary

- **Minimum:** one **`assets/icons/icon.png`** at **256×256** — used for the window, tray (auto 16×16), and (if set in build config) the built exe.
- **Optional:** **`assets/icons/icon.ico`** with 256, 48, 32, 16 px for sharper Windows exe/installer icons; then set `build.win.icon` to that path.
